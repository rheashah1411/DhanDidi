# Contributing to DhanDidi

Thank you for helping make financial documents easier to understand.

1. Fork the repository and create a focused branch.
2. Install development dependencies with `pip install -r requirements-dev.txt`.
3. Run `ruff check .` and `pytest` before opening a pull request.
4. Update all three languages when changing visible text.
5. Use only synthetic or fully redacted documents in tests and issues.

Do not commit Kaggle downloads, OCR caches, account details, tax identifiers, or other personal information. Explanations must remain educational and must not promise legal or financial outcomes.
